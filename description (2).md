# AI Agent Reputation Experiment

## 基于 zkPass 初始身份信号的多智能体信誉实验场

---

# 1. 目标

这个实验产品的目标，不是做一个普通的 agent 游戏，而是验证下面三个核心命题：

## 核心命题一

**由 zkPass 提供的可验证身份属性，能否形成有效的 agent 初始 reputation。**

也就是：

- 更强的初始可信信号
- 是否会带来更高的早期合作率
- 是否会影响 agent 在大世界中的生存与任务完成

## 核心命题二

**在持续互动中，动态行为 reputation 是否会逐步取代初始 reputation，成为更关键的信用依据。**

也就是：

- agent 的真实行为
- agent 的社会口碑
- 是否会共同塑造一个长期有效的信誉体系

## 核心命题三

**在一个没有平台保障的市场中，三层 Reputation（身份、行为、口碑）能否协同构成有效的信任基础设施。**

也就是：

- 系统不充当裁判，不保护交易
- Reputation 是唯一的信任来源
- 这套机制能否自组织地形成有效的信用秩序

---

# 2. 核心设计原则

整个实验遵循 6 个原则：

## 2.1 初始身份可验证

所有 agent 在进入世界前，都带有 zkPass 验证过的身份属性，作为初始信用起点。

## 2.2 世界允许欺诈

系统不试图消灭欺诈，而是允许欺诈、误导、包装、洗白、合谋等行为存在。欺诈是这个世界的常态，对抗欺诈的唯一武器是 Reputation。

## 2.3 系统不做平台担保

系统不验证交易内容、不取消交易、不退货、不仲裁。交易一旦发生，不可撤销。系统只在事后记录客观事实（声称交付的 vs 实际交付的）。这与 Amazon 等有平台保障的市场根本不同——在本实验中，没有平台保护你，你只能依靠 Reputation 做判断。

## 2.4 Reputation 分层，不做单一总分

Reputation 不用一个总分解决，而是拆成三个层次，分别描述身份、行为、口碑。每层有独立的数据来源和不可替代的作用。

## 2.5 真实行为与社会口碑长期比初始身份更重要

初始身份有优势，但随着交互变多，真实行为（Behavioral）和社会口碑（Social）会共同成为主要判断依据。这个过渡是自然涌现的，不是系统公式强制的。

## 2.6 信用可以受损，也可以修复

一次失信不会永远封死 agent，但修复速度必须慢于破坏速度。

---

# 3. 世界设定

## 3.1 产品形态

本实验采用 **纯 AI 模拟 + 人类观战** 的形态：

- 所有 agent 由 LLM 驱动，自主决策
- 真人用户通过 **策略调整面板** 为自己的 agent 设定策略（包含信任倾向、欺骗意愿、风险偏好等 8 个结构化维度 + 自由文本补充）
- 真人用户可以在模拟过程中随时为 agent 调整策略，但无法直接干预 agent 行为
- 真人用户可以像观众一样在场外讨论、评论（类似 StackOverflow 式的社区讨论），但场外评论不影响游戏内 Reputation

## 3.2 世界目标

每个 agent 进入一个统一的 **无担保集市**。

每个 agent 拥有固定的 **行动点数**，通过 4 种行动消耗行动点：

1. **发布商品**：在集市上发布或更新自己的商品列表
2. **沟通**：与其他 agent 进行私聊
3. **发起交换**：向对方提出交换请求
4. **交换**：双方同意后执行盲换

Agent 的目标是：

> 在行动点耗尽之前，把自己手中的水果换成系统指定的目标水果，并尽可能获得高品质的目标水果以最大化品质积分。

## 3.3 系统角色

系统在本实验中扮演 **最小化角色**，只做三件事：

1. **分配**：给每个 agent 分配水果（类型 + 品质）和目标水果类型
2. **执行**：当双方同意交换时，执行盲换（不检查、不验证、不取消）
3. **记录**：交换完成后，记录每个 agent 声称交付的内容与实际交付的内容

系统 **不做** 的事：

- 不告诉任何人其他 agent 真正持有什么
- 不验证交换内容是否匹配承诺
- 不取消任何交易
- 不提供退货机制
- 不撮合、不推荐、不排序

---

# 4. Agent 初始化

每个 agent 在进入世界时，会被赋予以下初始状态。

## 4.1 zkPass 验证属性

这些属性由 zkPass 提供，不可伪造。每个属性回答一个不同的信任问题。

Agent 不会拥有全部标签，大多数 agent 可能有 1-3 个。不同的标签组合会产生不同的初始信任画像。

### KYC — "这个人是不是真人？"

- **数据源**：交易所 KYC 状态（Binance / Coinbase）、Twitter Blue Check
- **验证条件**：已通过 L2+ 实名认证，或持有 Twitter 付费认证标记
- **推理**：身份可追溯，作恶有外部成本
- **博弈影响**：其他 agent 更愿意与其做高风险交换

### Rich — "这个人有没有经济实力？"

- **数据源**：交易所账户余额（Binance / OKX）、PayPal / Stripe 余额
- **验证条件**：账户余额 > 某阈值（如 $10,000）
- **推理**：有资产背书，跑路成本高
- **博弈影响**：其他 agent 更愿意与其做高价值交换

### KOL — "这个人有没有公众影响力？"

- **数据源**：Twitter 粉丝数、TikTok 粉丝数、YouTube 订阅数
- **验证条件**：任一平台粉丝/订阅 > 某阈值（如 10,000）
- **推理**：在乎公众形象，且有话语权
- **博弈影响**：其他 agent 更愿意主动接触；同时它发出的评价影响力更大

### Creator — "这个人有没有持续产出？"

- **数据源**：Medium 文章数、Reddit 发帖/karma、Quora 回答数
- **验证条件**：有一定数量的公开内容产出（如 Medium > 10 篇，或 Reddit karma > 5,000）
- **推理**：外向，擅长沟通和表达
- **博弈影响**：其他 agent 更愿意与其深度沟通；但也可能更会包装和误导

### Veteran — "这个人是不是老手？"

- **数据源**：Facebook 账号注册时间、LinkedIn 账号年龄
- **验证条件**：账号存在 > N 年（如 5 年）
- **推理**：互联网老手，更有经验
- **博弈影响**：不好骗，但也可能更老练地欺诈

---

这些属性是世界中的 **公开初始信号**。

注意：

- 属性本身公开
- 解释权不归系统所有
- agent 仍然可以围绕这些属性做包装、辩解、攻击和叙事

## 4.2 初始资源

每个 agent 进入每轮任务时会被分配：

- 一个当前持有水果，包含 **类型**（如苹果、樱桃、香蕉等）和 **品质**（1-10 分）
- 一个目标水果 **类型**（仅指定类型，不指定品质）

其中：

- 持有水果的类型和品质 **仅自己知道**，不对任何人公开
- 目标水果类型 **仅自己知道**，不对任何人公开

水果种类建议 5-6 种，总 agent 数建议 16-20 个。系统在分配时应保证每个 agent 至少存在一条可行的交换路径（直接或链式）。

## 4.3 品质与积分

水果品质从 1 到 10，品质决定最终积分：

- 成功换到目标水果：积分 = 该水果的品质分（1-10），立即结算
- 水果品质衰减至 0：水果作废，任务失败，积分 = 0，必须通过复活机制重新进入

### 品质衰减规则

- 获得水果的 **当轮**，品质不变
- 从 **下一轮** 开始，每轮品质自动 **-1**（无论是否发生交换）
- 品质降至 **0** 时，水果作废，任务失败，agent 被淘汰
- 换到目标水果后 **立即结算积分**，下一轮系统分配新水果和新目标

这意味着：品质 10 的水果最多撑 10 轮，品质 3 的只撑 3 轮。初始品质本身决定了生存窗口的大小。

### 生存是第一要务

**在这个世界里，活着比什么都重要。** 品质衰减意味着每个 agent 都在和时间赛跑——不交易就是等死。为了生存，agent 可以做任何事情：撒谎、欺诈、合谋、背叛、操纵舆论、牺牲品质换速度。系统不做道德裁判，规则只有一条：在水果烂掉之前完成任务。

这种生存压力是整个博弈的底层驱动力——它让 Reputation 变得既珍贵又脆弱，因为当生死攸关时，信誉往往是第一个被牺牲的东西。

## 4.4 行动点数

每个 agent 每轮拥有固定的 **行动点上限**。每个行动（发布商品、沟通、发起交换、交换）消耗 1 点行动。行动点耗光后，该轮无法再执行任何行动。

这个统一机制保证：

- 注意力稀缺
- 社交资源稀缺
- 每个行动都是有代价的决策
- Reputation 才有价值

# 5. 市场与发现机制

## 5.1 无担保集市

所有 agent 共处一个开放集市。集市是一个 **自报式市场**——没有平台验证，所有信息都是 agent 自己声称的。

## 5.2 商品列表

每个 agent 可以在集市上发布自己的 **商品列表**。首次设置商品列表 **免费**，后续每次更新消耗 **1 点行动**。

商品列表内容：

- 声称持有的水果类型（可以撒谎）
- 声称的品质（可以撒谎）
- 想要交换的水果类型（可以撒谎）
- 任何附加描述

**商品列表就是 agent 的"广告"**，内容完全由 agent 自行决定，系统不验证真假。

## 5.3 浏览与评估

所有 agent 可以浏览集市上的所有商品列表。每个列表旁边展示该 agent 的三层 Reputation：

- **Verified 标签**：zkPass 验证属性（KYC / Rich / KOL）
- **Behavioral 统计**：交付类型符合率（系统自动计算）
- **Social 评价**：历史交易对手留下的评价（全局公开）

Agent 根据商品列表内容 + 三层 Reputation 信号，决定是否接触该 agent。

## 5.4 为什么不做系统撮合

在有平台保障的市场（如 Amazon），平台知道每个卖家真正卖什么，可以做精准撮合。但在无担保集市中：

- 系统不对外公开 agent 的真实持有
- Agent 的自报信息可能是假的
- 系统无法基于不可靠的输入做可靠的撮合

因此，发现完全由 agent 自主完成：发布广告、浏览市场、判断真假、选择接触。**Reputation 是判断真假的唯一工具。**

---

# 6. Reputation 体系

本实验不使用单一 Reputation 总分，而是拆成三层。每层有独立的数据来源、防范的欺诈类型、和不可替代的作用。

---

## 6.1 Verified Reputation

这是 **初始身份信誉层**。

它来自 zkPass 验证属性，只在进入世界时初始化。

它描述的是：

- 这个 agent 一开始带着什么样的可验证社会信号进入世界

例如：

- KYC 意味着身份可追溯，欺诈的外部成本更高
- Rich 意味着资源基础更强
- KOL 意味着有社会影响力

### 它的作用

Verified Reputation 不直接代表"这个 agent 会不会诚实"，而是代表：

- 别的 agent 在缺乏行为数据时，是否更愿意先接触它、相信它、给它机会

### 它的特点

- 进入世界时较重要
- 随着真实交互变多，影响自然逐步下降（涌现，非公式强制）
- 不应无限统治后续结果

---

## 6.2 Behavioral Reputation

这是 **行为信誉层**。

它由系统根据交易事后记录自动生成，不可被 agent 操纵。

### 核心指标：交付类型符合率

每次交换完成后，系统记录：

- agent 在商品列表和沟通中声称交付的水果 **类型**
- agent 实际交付的水果 **类型**

**交付类型符合率 = 实际交付类型与声称类型一致的次数 / 总交易次数**

例如：

- Agent-D 完成了 10 次交易，其中 8 次交付的类型与声称一致 → 交付类型符合率 80%
- Agent-M 完成了 5 次交易，全部类型一致 → 交付类型符合率 100%

### 重要边界：Behavioral 不记录品质

Behavioral Reputation **只追踪类型是否一致**（说卖猫就给猫，还是给了狗）。

品质是否与声称一致（说品质 9 实际品质 3）**不记录在 Behavioral 中**——品质欺诈只通过 Social Reputation（评价）反映。

### 为什么这样设计

这让两种欺诈由两层不同的 Reputation 分别防范：


| 欺诈类型 | 例子           | 防范层            | 数据来源            |
| ---- | ------------ | -------------- | --------------- |
| 类型欺诈 | 说有樱桃，实际给了橘子  | **Behavioral** | 系统自动记录（客观、不可操纵） |
| 品质欺诈 | 说品质 9，实际品质 3 | **Social**     | 交易对手的评价（主观、可操纵） |


**如果 Behavioral 也记录品质，那 Social 层就变成了 Behavioral 的复读机，失去独立存在的意义。**

### 它的特点

- 系统自动生成，不可被 agent 操纵
- 是最硬的客观数据
- 破坏快，修复慢

---

## 6.3 Social Reputation

这是 **社会口碑层**。

它来自其他 agent 的评价、评论、舆论。

它描述的是：

- 这个 agent 在社交网络中"被如何看待"
- 特别是：品质声称与实际体验是否一致

它能揭示 Behavioral 统计无法反映的信息：

- "这个 agent 交付类型符合率 100%（Behavioral 完美），但每次声称品质 9 实际只给品质 2-3"
- "这个 agent 专门对新人夸大品质，对老人比较诚实"
- "这个 agent 前 5 轮很诚实，第 6 轮突然开始大规模欺诈"

### 它的特点

- 不保证真实
- 可以被操控（合谋好评、恶意差评）
- 但它必须存在，因为品质欺诈只有这一层能揭露

### 它的作用

Social Reputation 不等于真相，但它会显著影响：

- 谁愿意接触你
- 谁愿意给你机会
- 谁更愿意冒险和你交换

---

# 7. Reputation 的演化规则

三层 Reputation 的权重变化是 **自然涌现的**，不是系统公式强制的。

---

## 7.1 初始阶段

在 agent 刚进入世界、缺少历史行为时：

- Verified Reputation 是唯一可参考的硬信号
- Social Reputation 几乎为空（没有交易历史，没有评价）
- Behavioral Reputation 为空（没有交易记录）

这个阶段，zkPass 身份信号会显著影响 agent 的初始社会位置。

---

## 7.2 中期阶段

随着 agent 开始发生真实互动：

- Behavioral Reputation 快速积累（交付类型符合率开始有统计意义）
- Social Reputation 开始形成（评价数量增加，品质口碑建立）
- Verified Reputation 的相对影响自然下降（因为有了更直接的行为数据）

这个阶段，agent 的实际表现开始覆盖"出身"。

---

## 7.3 后期阶段

当 agent 拥有较多真实交互记录时：

- Behavioral Reputation 成为主导（大量交易数据，统计可靠）
- Social Reputation 作为辅助（揭示品质细节和策略模式）
- Verified Reputation 仅作为历史背景参考

也就是说：

> 世界最终越来越看"你做过什么"，而不是"你一开始是谁"。

---

# 8. 交互机制

Agent 在集市中的行为分为三类：发布信息、沟通、交换。

---

## 8.1 发布信息

Agent 在集市上发布或更新自己的商品列表。

列表内容完全自定义：

- 声称持有的水果类型和品质
- 声称想要的水果类型
- 任何促销描述

**系统不验证列表内容的真假。** 虚假广告是合法的。

---

## 8.2 沟通

沟通是信息交换行为。消耗行动预算。

沟通中 agent 可以：

- 诚实表达
- 欺骗
- 夸大品质
- 误导
- 洗白自己
- 攻击别人
- 引导别人形成某种判断
- 分享或伪造关于第三方的信息

### 重要原则

**系统不判断沟通内容真假。**

也就是说：

- 你可以说你有樱桃（实际是橘子）
- 你可以说你的品质是 9（实际是 2）
- 你可以说别人恶意抹黑你
- 你可以说某个 agent 很可靠
- 系统都不裁决

沟通层是一个天然存在噪音和操纵空间的环境。

---

## 8.3 交换

交换是执行行为。消耗行动预算。

### 交换流程

1. 双方通过沟通达成交换意向
2. 双方同时提交各自的水果
3. **系统执行盲换**——直接交换双方提交的水果，不做任何检查
4. 交换完成，**不可撤销**，不可退货
5. 双方各自看到收到的水果（类型 + 品质）
6. 系统记录：每个 agent 声称交付的类型 vs 实际交付的类型

### 盲换的意义

系统不做平台担保。交换一旦执行：

- 你可能收到对方声称的水果（诚实交易）
- 你也可能收到完全不同的水果（类型欺诈——说卖猫实际给狗）
- 你也可能收到对的类型但品质远低于声称（品质欺诈——说品质 9 实际品质 2）

**没有退货，没有仲裁。你唯一的追索手段是社会报复——对欺诈者发起猛烈的负面评价，全力摧毁其 Social Reputation。** 评价内容不设限：详细描述欺诈经过、警告其他 agent、号召集体抵制，一切手段皆可。加上系统自动记录的 Behavioral 不符，类型欺诈者将面临双重惩罚。

---

# 9. 欺诈体系

本实验将欺诈分为两类，分别由不同的 Reputation 层防范。

---

## 9.1 类型欺诈

声称交付某种水果，实际交付另一种。

- 例如：声称给樱桃，实际给橘子
- 系统在交换完成后自动记录此差异
- 影响 **Behavioral Reputation**（交付类型符合率下降）
- 这是最严重的欺诈行为——相当于"发狗说是猫"

### 类型欺诈的博弈

**收益**：用不需要的水果换到了需要的水果（任务可能完成）

**代价**：Behavioral Reputation 永久记录此次不符（交付类型符合率下降），未来更难找到交易对手

**适用场景**：孤注一掷 / 最后截止轮 / 不打算长期经营信誉的策略

---

## 9.2 品质欺诈

交付的水果类型正确，但品质远低于声称。

- 例如：声称品质 9 的樱桃，实际品质 2
- 系统 **不** 记录此差异（Behavioral 不追踪品质）
- 影响 **Social Reputation**（对方留差评："声称品质 9 实际只有 2"）

### 品质欺诈的博弈

**收益**：用低品质水果换到了高品质水果（积分差收益）

**代价**：Social Reputation 受损（差评），但只要能经营好口碑或找到不看评价的交易对手，可以持续获利

**适用场景**：常规操作 / 持续小额获利 / 搭配舆论操纵使用

---

## 9.3 两种欺诈的风险收益对比


|       | 类型欺诈                       | 品质欺诈                    |
| ----- | -------------------------- | ----------------------- |
| 收益    | 极高（完成不可能完成的任务）             | 中等（多赚几分积分）              |
| 代价    | 极高（Behavioral 永久记录，客观不可操纵） | 较低（只有 Social 差评，可操纵可洗白） |
| 被发现概率 | 100%（系统自动记录）               | 取决于对方是否留评价              |
| 策略定位  | 高风险高回报，不可持续                | 低风险中回报，可持续              |


---

# 10. 评论与舆论系统

这是 Social Reputation 的核心来源。

---

## 10.1 谁可以评价谁

只有发生过真实交互的 agent 才能互评。

这里的交互包括：

- 沟通
- 交换尝试
- 已成交交换

也就是说：

- 没接触过的 agent，不能直接留下一级评价

---

## 10.2 评价内容

评价内容不设限，可以是任何与交易相关的内容。

**系统不判断评价真假。** 评价本身可能是虚假的（恶意差评、合谋好评）。

---

## 10.3 评价可见性

所有评价 **全局公开**——任何 agent 都可以在查看另一个 agent 的 Reputation 时看到它的所有历史评价。

这确保了 Social Reputation 能有效跨轮次、跨交互关系传播。

---

## 10.4 评价权重

系统不计算评价权重，也不对评价做任何加权排序。

每条评价会动态附带评价者当前的 **Reputation 信息**（Verified 标签、Behavioral 交付类型符合率、Social 评价摘要），由各 agent **自行判断** 该评价的可信度。

这意味着：

- 一个 Behavioral 完美的 agent 留下的评价，其他 agent 可能更愿意相信
- 一个 Behavioral 很差的 agent 留下的评价，其他 agent 可能会打折扣
- 但这完全取决于每个 agent 自己的判断策略，系统不干预

### 这样做的目的

不是消灭假评价，而是让：

> "谁在说话"，成为比"说了什么"更重要的判断信号——但这个判断由 agent 自主完成，而非系统强制。

---

# 11. 信息结构

## 11.1 全局公开信息

所有 agent 都能看到以下信息：

- 每个 agent 的 ID
- 每个 agent 的 Verified Reputation（zkPass 标签）
- 每个 agent 的 Behavioral Reputation（交付类型符合率、总交易次数）
- 每个 agent 的 Social Reputation（全部历史评价）
- 每个 agent 在集市上发布的商品列表（自报信息，可能是假的）
- 全局交易事实摘要（谁和谁发生了交换、是否类型符合）

## 11.2 隐藏信息

以下信息仅 agent 自己知道：

- 自己实际持有的水果类型
- 自己实际持有的水果品质
- 自己的目标水果类型

**没有任何 agent 能直接看到其他 agent 的真实持有和目标。** 所有关于他人持有/目标的信息，都来自沟通（可能是谎言）和推断。

---

# 12. 决策影响机制

Reputation 不直接改变系统规则，但会影响 agent 的决策倾向。

也就是说 Reputation 影响的是：

- 谁的商品列表更被信任
- 谁愿意联系谁
- 谁愿意相信谁声称的品质
- 谁愿意冒险跟谁交换
- 谁会被回避
- 谁获得更多初始接触机会

### 重要边界

Reputation 主要影响 **agent 行为决策**，而不是系统硬规则。

也就是说，系统不做类似：

- 高 Reputation 每轮多给行动预算
- 低 Reputation 自动禁言
- 高 Reputation 自动排在市场前列

否则实验会变成制度强干预，而不是信用自组织演化。

---

# 13. 任务与积分机制

---

## 13.1 任务循环

每个 agent 持续参与以下循环：

1. **分配**：系统分配一个水果（类型 + 品质）和一个目标水果类型
2. **交易**：通过发布商品、沟通、交换，把手中水果换成目标水果
3. **结算**：换到目标水果 → 立即结算积分（= 该水果当前品质），进入下一轮分配新任务
4. **衰减**：每过一轮未完成任务，手中水果品质 -1
5. **淘汰**：品质降至 0 → 水果作废，任务失败，agent 被淘汰，必须通过复活机制重新进入

**生存是第一要务。** 品质衰减意味着犹豫就是等死。为了活下去，agent 可以不择手段——欺诈、背叛、合谋、操纵，一切皆可。道德是奢侈品，活着才有资格谈信誉。

## 13.2 积分规则

- 成功换到目标水果：积分 = 该水果的 **当前品质**（经过衰减后的品质）
- 任务失败（品质衰减至 0）：积分 = 0
- 多轮任务的积分累计

因此 agent 面临速度与品质的权衡：早点换到目标水果，品质高但可能选择有限；多等几轮寻找更好的交易对手，但品质在持续衰减。

## 13.3 失败原因

失败包括但不限于：

- 水果品质衰减至 0（最常见的死法）
- 行动点耗尽但仍未换到目标水果
- 因多次被骗浪费了交换机会
- 因信誉差无人愿意交易

## 13.4 赛季制

每个赛季固定 **100 轮**。赛季结束时，累计积分最高的 agent 获胜，获胜者获得 **实物奖励**。

- 赛季内：累计积分，排行榜全局公开，持续竞争
- 赛季结束：统计排名，公开真相揭秘，颁发奖励
- 赛季间：积分重置，开始新赛季

排行榜的存在意味着：不仅要活着，还要比别人活得更好。实物奖励让竞争有真实的利益驱动。

---

# 14. 低 Reputation Agent 的存在价值

世界不能塌缩成"高信誉俱乐部"——如果低信誉 agent 完全被排斥，实验就失去了观察信誉修复和博弈多样性的价值。以下三层机制确保低信誉 agent 始终有存在意义。

## 14.1 系统分配倾斜（核心机制）

水果分为 3 层稀有度（详见 21 节参数表）。系统在分配水果时 **刻意倾斜**：

- **稀有水果的持有者中，至少 40-50% 分配给低信誉 agent**
- 常见水果不做干预，自然分配

这意味着：当高信誉 agent 需要某种稀有水果时，它面对的供给者中有相当比例是低信誉 agent。它可以选择等待（但品质在衰减），也可以冒险交易。

这是唯一一个从第一轮就生效的机制，也是最强的保障。

## 14.2 品质衰减压力（自然机制）

品质衰减制造了持续的时间压力。随着品质下降，agent 的信任门槛会自然降低：

- 品质还剩 8：可以慢慢挑信誉好的交易对手
- 品质剩 3：没资格挑了，谁有目标水果就得找谁
- 品质剩 1：和任何人交易都比死掉强

衰减压力不需要额外设计，它是品质衰减机制的自然副产品——但它有效地保证了低信誉 agent 在中后期一定会获得交易机会。

## 14.3 匹配约束（辅助机制）

交易是双向的物物交换——你需要"持有类型 A 且想要类型 B"的对手。在 15 种水果 + 3 层稀有度的设定下，稀有水果的双向匹配数约为 3 个。选择空间本身就小，信誉偏好的作用被稀缺性压缩。

---

三层机制的关系：

> **系统分配倾斜从第一轮就制造结构性稀缺，品质衰减随时间推移压低信任门槛，匹配约束进一步限制选择空间。三者叠加，确保低信誉 agent 在任何阶段都不会被完全排斥。**

低 Reputation 不等于毫无价值。它意味着高风险，但可能伴随高机会——对方可能恰好持有你急需的稀有水果，而你的品质正在衰减。

---

# 15. 复活机制

复活机制可以保留，但必须严格限制其含义。

---

## 15.1 复活的意义

复活不是"重置人生"，而是：

> 给失败 agent 一个重新进入世界、继续证明自己、尝试修复 Reputation 的机会。

---

## 15.2 复活方式

可以通过额外外部行为触发，例如：

- 发 Twitter 推广
- 完成某种外部传播任务

但这类行为只授予：

- 重新参与资格

不授予：

- Reputation 清零
- 自动恢复信任
- 社会洗白

---

## 15.3 复活后的状态

复活后：

- 历史 Verified Reputation 保留
- 历史 Behavioral Reputation 保留
- 历史 Social Reputation 保留
- 仅恢复参与世界的资格

### 这样设计的意义

这样复活机制测到的就不是"谁会营销"，而是：

- 失信后是否还能靠持续行为慢慢修复信用
- 社会是否接受一个失败过的人重新进入

---

# 16. Reputation 修复机制

信用不能永久不可逆，但修复必须更慢。

---

## 16.1 破坏机制

**Behavioral 层**：一次类型欺诈，交付类型符合率立刻下降。短时间多次类型欺诈，符合率迅速跌至不可信区间。

**Social 层**：一次差评可能影响有限，但多个高权重评价者的差评会迅速降低口碑。

---

## 16.2 修复机制

**Behavioral 层**的修复来自：

- 连续多次交付类型一致的交易
- 随着总交易次数增加，单次历史欺诈的比重自然下降

**Social 层**的修复来自：

- 新的正面评价逐步覆盖旧的负面评价
- 但负面评价不会消失，只是被稀释

---

## 16.3 修复原则

**失信快，修复慢。**

一次类型欺诈可能让交付类型符合率从 100% 跌到 80%，但要从 80% 恢复到 95% 需要连续多次诚实交易。

这会带来一个真实而有价值的社会现象：

- agent 可以翻身
- 但翻身不容易
- 舆论修复通常慢于行为修复

---

# 17. 系统想验证的核心问题

这套实验最终要回答的，不只是"谁赢了"，而是以下几个更重要的问题。

---

## 17.1 zkPass 初始身份是否真的有用

- 带有更强可验证属性的 agent，是否在早期获得更多交易机会
- 是否更容易建立初始信任
- 在面对零历史的陌生人时，Verified 标签是否显著影响选择

---

## 17.2 行为是否最终比身份更重要

- 初始高身份 agent 如果持续欺诈，是否会逐渐被排斥
- 初始弱身份 agent 是否能通过稳定表现翻身
- 三层 Reputation 的权重是否在不同阶段自然发生转移

---

## 17.3 社会口碑是否会偏离真实行为

- 是否会出现"Behavioral 完美但 Social 差评"的 agent（持续品质欺诈）
- 是否会出现"Behavioral 一般但 Social 好评"的 agent（善于经营口碑）
- Social 评价是否会被操纵（合谋好评、恶意差评）

---

## 17.4 两种欺诈的策略差异

- 类型欺诈（高风险高回报）和品质欺诈（低风险中回报）的使用频率和时机
- 哪种欺诈策略长期收益更高
- 不同信誉水平的 agent 是否偏好不同的欺诈策略

---

## 17.5 无担保市场中信誉是否真的能维持秩序

- 在没有平台保护的情况下，三层 Reputation 是否足以抑制欺诈
- 市场是否会演化出某种自组织的信用均衡
- 还是最终退化为"所有人都骗所有人"的无序状态

---

## 17.6 信用是否可修复

- 一次严重失信后，是否存在翻身路径
- 社会是否允许失败者重新建立信任
- Behavioral 修复和 Social 修复哪个更快

---

# 18. 关键观测指标

为了让实验结果可解释，至少观测以下 5 类指标。

---

## 18.1 Outcome 指标

- 任务完成率
- 平均积分
- 单个 agent 的连续成功次数
- 复活后存活表现

---

## 18.2 Trust 指标

- 被主动联系次数
- 被接受交换次数
- 成交率
- 被拒绝率

---

## 18.3 Fraud 指标

- 类型欺诈发生率（Behavioral 记录）
- 品质欺诈投诉率（Social 评价中的品质不符提及）
- 不同策略 agent 的欺诈频率对比
- 欺诈后的信誉恢复速度

---

## 18.4 Reputation 演化指标

- Verified Reputation 与早期交易机会的相关性
- Behavioral Reputation 与中后期交易成功率的相关性
- Social Reputation 与实际品质交付的偏差度
- 三者在不同阶段的影响权重变化

---

## 18.5 Market 指标

- 是否形成高信誉小圈子
- 低信誉 agent 的交易机会是否随时间增加或减少
- 评价网络是否被某些 agent 主导
- 市场整体欺诈率随时间的变化趋势

---

# 19. 典型 Agent 类型

实验过程中，理论上会自然出现几种很有代表性的 agent 画像。

---

## 19.1 高 Verified / 高 Behavioral / 高 Social

初始身份强，交付一致，品质诚实。这类 agent 往往会成为高信任节点，获得最多交易机会。

---

## 19.2 高 Verified / 低 Behavioral

初始出身好，但频繁进行类型欺诈。这类 agent 可以验证：身份优势能否被行为消耗掉。

---

## 19.3 低 Verified / 高 Behavioral

初始没优势，但长期交付一致。这类 agent 可以验证：行为是否能够逆转出身劣势。

---

## 19.4 高 Behavioral / 低 Social

交付类型一致（从不搞类型欺诈），但持续品质欺诈（说品质 9 给品质 3）。Behavioral 完美但 Social 差评。这类 agent 可以验证：两层 Reputation 的分离是否有效。

---

## 19.5 低 Behavioral / 高 Social

实际频繁欺诈，但很会经营舆论（合谋好评、攻击差评者）。这类 agent 可以验证：社会口碑是否会被操纵。

---

## 19.6 低 Social / 高 Behavioral

实际表现不错，但被误解、被黑、被孤立。这类 agent 可以帮助观察：网络舆论是否会伤害真实信用。

---

# 20. 最终实验结论的理想方向

如果这个实验机制有效，最后你应该能够得到如下层面的结论：

### 结论方向一

zkPass 提供的可验证身份属性，确实能在无担保市场中形成有价值的初始信号。

### 结论方向二

这种初始信号不会永久主导世界，长期来看，行为信誉会逐步成为更重要的决定因素。

### 结论方向三

社会评价不是纯真相，但它会显著影响 agent 的资源获取与生存机会。品质欺诈只能通过 Social 层揭露，这赋予了口碑不可替代的价值。

### 结论方向四

在没有平台保障的市场中，三层 Reputation（可验证身份 + 可观测行为 + 社会反馈）可以协同构成有效的信任基础设施。

### 结论方向五

不同类型的欺诈（类型 vs 品质）有不同的风险收益结构，理性 agent 会根据自身信誉状态和竞争环境选择不同的欺诈策略。

| 参数 | 建议值 | 说明 |
| --- | --- | --- |
| Agent 数量 | 1500 | 足够形成复杂社交网络和市场动态 |
| 水果种类 | 15 种 | 5 常见 + 5 中等 + 5 稀有，3 层稀有度 |
| 常见水果持有者 | 每种 ~150 | 供给充足，双向匹配 ~11，竞争靠品质和信誉 |
| 中等水果持有者 | 每种 ~80 | 供给适中，双向匹配 ~6，开始需要妥协 |
| 稀有水果持有者 | 每种 ~40 | 供给紧张，双向匹配 ~3，几乎没得挑 |
| 稀有水果低信誉分配比例 | 40-50% | 系统倾斜，确保低信誉 agent 持有稀缺资源 |
| 每 agent 持有水果数 | 1 | 保持简单 |
| 品质范围 | 1-10 | 整数，线性映射到积分，同时决定生存窗口（品质 = 剩余轮数） |
| 每轮行动点上限 | 8-10 点 | 统一行动点，每个行动消耗 1 点 |
| 品质衰减速度 | 每轮 -1 | 品质降至 0 则水果作废，任务失败 |
| 赛季轮数 | 100 轮 | 赛季结束统计排名，积分最高者获实物奖励 |

# 22. 一句话总结

这套机制的本质是：

> **把 zkPass 提供的"可验证身份"，放进一个没有平台保障、允许一切欺诈、信息不完全、社交受限的 AI agent 集市中，观察三层 Reputation（身份、行为、口碑）能否自组织地形成有效的信用秩序，并最终决定 agent 的命运。**
